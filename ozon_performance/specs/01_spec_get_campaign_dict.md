# Spec for get_campaign_dict

## Summary

Публичная функция в `ozon_performance.py`, возвращающая справочник всех рекламных кампаний аккаунта — уникальные сущности с атрибутами, без дат.

Реализация уже присутствует в скаффолде в полном виде (не стаб). Шаг 4 для этой функции — верификация с реальным API и фиксация результата.

## Functional Requirements

### 1. Сигнатура функции

```python
def get_campaign_dict() -> pd.DataFrame:
```

Параметров нет — возвращает все кампании аккаунта.

### 2. Алгоритм

Синхронный паттерн:
- `GET /api/client/campaign` без параметров
- Ответ: `{"list": [{id, title, state, advObjectType, fromDate, toDate, dailyBudget, ...}]}`
- Разбираем массив `list`, маппируем поля → строки DataFrame
- Дропаем строки без `campaign_id`, дедуплицируем по `campaign_id`
- Возвращаем с фиксированным порядком колонок `CAMPAIGN_DICT_COLUMNS`

### 3. Возвращаемый DataFrame

| Колонка | Тип | Источник в API | Описание |
|---------|-----|----------------|----------|
| `campaign_id` | string | `id` | Уникальный ID кампании — используется как ключ в запросах статистики |
| `campaign_name` | string | `title` | Название кампании |

### 4. Изменения в `ozon_performance.py`

Изменений не требуется — функция полностью реализована в скаффолде:
- Константа `CAMPAIGN_DICT_COLUMNS` уже есть
- `_fetch_all_campaigns()` уже реализован
- `get_campaign_dict()` уже реализован (не стаб)

## Ограничения API

| Ограничение | Значение | Реализация |
|-------------|----------|------------|
| Запросов в сутки | макс **100 000** | общий лимит аккаунта |
| 429 backoff | старт 10 сек, удвоение | `_get()` наследует через `_headers()` + session |

Метод `GET /api/client/campaign` — синхронный, без batching и UUID. 429 маловероятен, но `_session` с retry-логикой уже настроен в `_post()`. Для `_get()` дополнительный retry не нужен — при 429 на этом endpoint перезапустить вручную.

## Possible Edge Cases

- `id` отсутствует в объекте кампании — строка дропается через `dropna(subset=["campaign_id"])`
- `title` отсутствует — `_pick(c, "title")` вернёт `None`, колонка будет содержать `NaN`
- Аккаунт без кампаний — `_fetch_all_campaigns()` вернёт `[]`, функция вернёт пустой DataFrame с правильными колонками
- Ответ API — сразу список (не dict с ключом `list`) — обрабатывается в `_fetch_all_campaigns()` через `isinstance(data, list)`
- Дубликаты `campaign_id` в ответе — дедуплицируются через `drop_duplicates(subset=["campaign_id"])`

## Acceptance Criteria

- [ ] `get_campaign_dict()` возвращает DataFrame с колонками `["campaign_id", "campaign_name"]`
- [ ] Каждая строка — уникальная кампания (`campaign_id` без дублей)
- [ ] `campaign_id` не содержит `None` / `NaN`
- [ ] При пустом аккаунте возвращается пустой DataFrame с правильными колонками
- [ ] `df.shape[0] > 0` — хотя бы одна кампания в реальном аккаунте
- [ ] Тип колонки `campaign_id` — string (не int)
